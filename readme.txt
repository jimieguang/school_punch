最终更新代号LAST ORDER

本是兴趣使然写的打卡程序，不知不觉竟然维护了一年半之久
从开始的selenium模拟键鼠操作，到后来的直接发送post请求，到此暂且是代码洁癖导致的优化

然而学校又增加了滑块验证码作为阻碍（我不理解一个形式主义的鬼东西有什么必要
于是不得不再次使用臃肿的selenium获取验证码，曾尝试过js逆向解析validation参数，时间长达数周，但成果寥寥
期间学校神奇的脑回路又增加了填写三次体温的需要
笑
要求下午三点前必须提交的健康打卡却要填写当日早中“晚（强调）”的体温
于是在与学校的斗法中程序不断完善，增加了nonebot搭建的qq机器人推送，重构封装了函数，进行了多线程优化等

由于我的理念向来是将用户当作傻瓜（无冒犯意），因此为方便众人打卡，将本就复杂的维护任务变得更加繁琐
随着时间的流逝，打卡人数也越来越多，于是学校就整了封禁IP这一出幺蛾子
ps：似乎为了这一个小小功能，学校服务器崩了两天，影响了包括电费校园卡充值等真正关乎民生的需求，不愧是“以人为本”的您

当然，封禁IP本就是反爬措施的常见手段，也很容易利用代理池等手段进行破解，然而我放弃了

很抱歉，维护了这么长时间的项目说放弃就放弃了
说说当时的幼稚想法吧
其实一方面是想节省下这些无意义被浪费的时间
另一方面是想从另一个角度逼迫学校知道自动打卡程序的存在，借被邀请喝茶的机会尝试申请取消这个无意义的打卡
这也是为什么程序设计之初就从未试图隐藏过自己的真实IP，只有学校方面想，完全可以精确到宿舍号找到本人
但他没有
当打卡程序被发现时，我既没有被请去喝茶（ps：从github上看应该不止我这一批在做类似事情），学校也没有反思打卡存在的意义
反而是封堵的愈发严密，手段也愈发狠毒

我的放弃，一方面是志不在此，想要将时间投入到更加有意义的事情上去
另一方面则是看不到希望，与学校的攻防斗法没有任何意义，没有任何未来，即使继续维护，他们也会继续封堵
我也差不多该知晓了，打卡存在的意义不过是担责系统罢了，学生的权力在哪里呢？沉默的大多数中，又要求谁去发声呢？

最后，讲一下程序该如何使用吧，趁他还没有再一次封堵
将学号 密码 与昵称填入users.txt中，用两个空格分开(都在一行)，运行main.py文件即可
